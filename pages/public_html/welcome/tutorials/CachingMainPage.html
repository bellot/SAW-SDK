Caching main page

If we consider the design of the main page described in Main page design, we can see that the page structure does not use Transaction variables or Session environment. Thus, if, for a given page, the content of the main div does not use Transaction variables or Session environment, then this page can be cached. That is why we propose two alternative classes for implementing the main page:

  • WebTechPageWithUpdatableMemoryCache which caches the page content in memory ; it has two constructors: 
        public WebTechPageWithUpdatableMemoryCache(ElementInterface mainDivElement) 
        throws Exception ;
    public WebTechPageWithUpdatableMemoryCache(String cssFile, ElementInterface mainDivElement) 
        throws Exception ;
  • WebTechPageWithUpdatableFileCache which caches the page content in a disk file. 
        public WebTechPageWithUpdatableFileCache(ElementInterface mainDivElement) 
        throws Exception ;
    public WebTechPageWithUpdatableFileCache(String cssFile, ElementInterface mainDivElement) 
        throws Exception ;

Usage. These two classes are to be used like the WebTechPage as explained in Main page design.

Example. If the main div contains only a "Hello world !" inside a H1 tag, then the page can be cached because it does not use Transaction variables or Session environment.
