SAW Utilites

This section is about:

  1. The utilities, programs that are part of the distribution.
  2. Some of the packages distributed with SAW that can be useful for a Web application developers.

Utilities

The following utilities are distributed with SAW. They are programs that maybe useful for the Web application developers or maintainers.

  • List backups [ mvn exec:java -P listbackups in directory SAW-SDK/pages ]
    When the server is running, it loads the Entity Binz from the xorkspace directory. At the shutdown of the server, the Entity Binz are saved in the same directory. However,one must be afraid of a bug or a power failure that would the save process or let the workspace corrupted. That's why the server, at regular interval of time, save the ENtity Binz in the workspace and copy the workspace directory in the backup and the name is prefixed by the current time in millis. The List backups utility simply list the backups with the time being written in clear, the name of the backup and the time the backup lasts from now. Backups are listed from the olderto the younger. Example of output:

    This may be useful to restore a backup as explained in the Restroring a backup of the workspace tutorial section.

  • Encrypt a password [ mvn exec:java -P encryptpassword in directory SAW-SDK/pages ]
    A Web application may have users with logins and passwords. Logins and passwords are stored in the org.saw.users.User data structure. We strongly suggest that the stored password must be encrypted. At the bootstrap phase, some kind of root user must be created with its password. Thus, we need the encryption of its password to initialize its data as explained in Initializing the admin suer tutorial section. The simple program Encrypt a password allows you to enter a password and get the encrypted form.

Useful packages

This section describes classes and packages that could be used by the Web application developer.

  • Package org.saw.util.beautifiers:
    • JavaBeautifier.java: converting Java file into HTML file in default Eclipse style.
  • Package org.saw.util.bytes:
    • BytesArray.java: C-like functions (strchr, strstr, etc.) working on byte arrays.
    • FileBytes.java: reading a file in a byte array.
    • GzipBytes.java: to gzip a byte array.
  • Package org.saw.util.checks:
    • JMAAADateCheck.java: checks if a string is a date in format J/M/AAAA.
    • PowerOfTwoCheck.java: checks if an int is a power of 2.
  • Package org.saw.util.compacters:
    • CssCompacter.java: to compact a CSS file.
    • HtmlCompacter.java: to compact a HTML file.
    • JsCompacter.java: to compact a JavaScript file.
  • Package org.saw.util.dates:
    • DatesUtils.java: a few functions on dates.
  • Package org.saw.util.encoding:
    • Base64Coder.java: a base64 encoder.
    • PasswordEncrypt.java: password encryption algorithm (MD5 + base64).
  • Package org.saw.util.files:
    • DirCopy.java: to copy a directory.
    • DirRm.java: to remove a directory.
    • FileCopy.java: to copy a file.
    • FileToString.java: to read a text file in a string.
    • FilesConcat.java: to concatenate several files in one.
    • GZipFile.java: to zip a file.
    • StringToFile.java: to write a string in a file.
  • Package org.saw.util.logs:
    • Logs.java: the logging functions of SAW.
  • Package org.saw.util.logs:
    • Logs.java: the logging functions of SAW.
  • Package org.saw.util.random:
    • RandomExt.java: to generate a random numerical if between 100.000 and 999.999.
  • Package org.saw.util.structures:
    • IntArray.java: extensible array of int, similar to ArrayList.
    • IntObjHashTable.java: hast table where keys are int and value are objets, inner table size is fixed.
    • ObjArray.java: extensible array of objects, similar to ArrayList.
    • ObjObjHashTable.java: hast table where keys and value are objets, inner table size is fixed.
  • Package org.saw.util.updatable:
    • UpdatableFriends.java: SAW version of the Observer in the Observer-Observable design pattern.
