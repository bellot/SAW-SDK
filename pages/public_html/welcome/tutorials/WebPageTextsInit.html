Initializing the Entity Binz

To initialize the WebPageTexts Entity Binz that we have designed in The Web page texts, we will use the bootstrap process. First, we program a class WebPageTextsBootStrap with a static method that will do the job: 

package org.site.conference.web;

/** @author  Patrick Bellot, © 2010 and later. */

public class PageTextsBootStrap 
{
    public final static void booStrap()
    {
        PageTexts pageTexts = PageTexts.entity ;
        
        pageTexts.set("WebTech 2012",
                      "2nd Annual International Conference on",
                      "Web Technologies & Internet Applications (WebTech 2012)",
                      "Special Track: Social Computing and Behavioral Modeling (SocialComp 2012)") ;
    }
}

And we call this method from the class org.site.bootstrap.Bootstrap: 

...

import org.site.conference.web.PageTextsBootStrap;

...

    public final static void bootStrap()
        throws Exception
    {
        initRootUser() ;
        
        PageTextsBootStrap.bootStrap() ;
    }

...

Then, be sure that the server is not running and in SAW-SDK/pages, run the Maven command: 

maven exec:java -P bootstrap

Warning. If you have already created the admin (root) user as explained in Initializing the admin user tutorial, an error will occur if you try to crete it once again. Two solutions:

  1. Comment the instruction initRootUser() ; in the code above.

  2. Erase the file SAW-SDK/workspace/org/saw/users/UsersCatalog/binz.

Note. We decided to put the bootstrap code in separate static classes so that these classes are not loaded when running the server. They are only loaded when the bootstrap process is run.
