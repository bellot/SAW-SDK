The bootstrap process

This is a general description of the bootstrap process. Examples will be given during the tutorials.

The bootstrap process allows to initialize or reinitialize the Entity Binz. Thus, we can use it to initialize the UsersCatalog described in The Users Catalog tutorials. The code to be executed during the bootstrap process must be written in the class org.site.bootstrap.Bootstrap in the method: 

public static final void bootStrap()
    throws Exception ;

This code can be run in SAW-SDK/pages with the Maven command: 

mvn exec:java -P bootstrap

Warning. As the bootStrap() method usually create or modify the Entity Binz, the server must not be running during a bootstrap run. Otherwise, at the shut down of the server, current server Entity Binz will overwrite the Entity Binz.

About Entity Binz. Entity Binz are save in the workspace directory under SAW-SDK/pages. For instance, the UsersCatalog Entity Binz which class is org.saw.users.UsersCatalog is saved in org/saw/users/UsersCatalog under the name binz. When running the boostrap process, existing Entity Binz are loaded in memory with their current states. Thus, if the programmer wants to create a totally new UsersCatalog, then he/she must erase the saved binz in org/saw/users/UsersCatalog.
