XEC and Transaction variables

In Using the Transaction variables, we have seen that we can access the Transaction variables. We can use these variables to check the input values of a Session Binz or to output them in the HTTP Response content.

We can incorporate the value of a Transaction variable in a XML Element by using the Dynamic Element of class TransactionVariable. This type of element can be included in the XML description of a page. The difference with ordinary XML elements is that its output is dynamically computed at the time of output. This dynamic element has a constructor: 

public void TransactionVariable(String variableName) ;

taking the name of a Transaction variable as argument. When it is executed, it produces as output the value of the varible.

Example. The class test/PHPTV.java is an example of using the TransactionVariable dynamic element. The result can be seen by providing a GET variables at test/XECTV.class?person=bellot.

As we have told before the TransactionVariables object has put methods. We can use these methods to artificially create or modify Transaction variables.

Example. The class test/PHPTV2.java use one of the put methods of TransactionVariables to artifcially set a Transaction variable named "place". A TransactionVariable dynamic element with variable name "place" is used in the XEC description of the page. The result can be seen by providing a GET variables at test/XECTV2.class?person=bellot.
