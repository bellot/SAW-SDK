The User data structure

In a Web application, there usually exists users who can log in and use application services according to user's right. The class User defines the minimum information about a Web application user. All the User objects are stored in an Entity Binz of class UsersCatalog. See next tutorial: Users Catalog.

User's information. These minimum user's information are:

  • The user id, a unique numerical identifier for the user of type int. It is automatically allocated by the function: 

    public final synchronized int createUser(String loginName, String encrypted password) ;
        throws LoginNameInUseException, NoFreeUserIdException ;

    of the class UsersCatalog with the static object UsersCatalog.entity.

  • The login name of type String. It uniquely identify the user at login time. We recommand using the user e-mail address as a password.

  • The encrypted password which is the user password encrypted with the function encryptof the class PasswordEncrypt.java.

  • The privileges field of type int. This field can be easily used by the Web application to store 31 cumulative user's rights.

Modification of information. Only user id is final and cannot be modified. The other fields can be modified with the setters of class User except the login name that must be modified using the function: 

public final synchronized void setLoginName(User user, String loginName)
        throws LoginNameInUseException ;

of the class UsersCatalog with the static object UsersCatalog.entity  because the login name must remain unique amongst users).

Additional information. The user's information are the minimum for handling users login and rights. However, the Web application may require additional information such as first name, last name, address and so on. Instead of modifying the User data structure, we strongly recommend to implement a complementary data structure as explained in a later tutorial.
