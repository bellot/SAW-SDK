The Session environment

When a client connect to a SAW server, a session identifier (sessionId) is allocated. The sessionId uniquely identifies the session. This sessionId is tranferred from the server to the client and vice versa using cookies. The sessionId identifies a Session object environment accessible to the Session Binz. From one client request to another, the same Session environment is provided to the Session Binz. It is similar to PHP session variables.

The Session environment object. The Session environment of class SessionEnvironment contains:

  • the session identifier ;

  • the IP address of the client as a string 

  • the User object if the user is logged, it is the responsability of the login and logout Session Binz to maintain this field 

  • a SessionVariables object that is accessible to the Session Binz to store values.

The Session variables object. The Session variable object is simply a hash table where keys are String and values are any kind of object. The methods for accessing the values are those of the class ObjObjHashTable.

How to access the Session environment ? The Session environment can be obtained from the Transaction object using the method: 

public final SessionEnvironment getSessionEnvironment() ;

The Session variables can be obtained from the SessionEnvironment object using the method: 

public final SessionVariables getSessionVariables() ;
