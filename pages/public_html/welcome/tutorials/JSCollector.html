Collecting JavaScript

As satted in Collecting CSS, a wellknown tip to avoid slowing the Web page delivery and rendering is to avoid too much external links such as external CSS or JavaScript files. If a Web application has many CSS files, the programemrs have to collect them into a single file. But a single file is hard to maintain.

The solution. SAW proposes a Session Binz able to collect several distinct JavaScript files and to deliver them as if it was a single JavaScript file. There exist two versions of this JavaScript Collector Session Binz. The first version, JsMultiMemorySessionBinz, caches the resulting JavaScript in memory for a quick delivery. And the second version, JsMultiFileSessionBinz uses a disk file for caching.

Usage. The usage of JavaScript Collector Session Binz is exactly the same as that of the CSS Collector Session Binz described in Collecting CSS.
