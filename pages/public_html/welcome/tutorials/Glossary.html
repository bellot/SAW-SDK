Glossary

Session environment. When a user agent connects to the server, the server attributes a Session identifier to it. This is a randomly generated integer value that is associated to the user agent using a cookie. This Session identifier uniquely determines a data structure called a Session environment. The Session environment is maintained on the server and available to the Session Binz answering a HTTP Request from this client. It is a statefull (remanent) object delivered by the Transaction Container. The type of the Session environment is SessionEnvironment. Initially it contains the Session identifier, the IP address of the client, a reference to a user of type User. if it has been set by the Web application after a login for instance. It also contains a SessionVariables. object. This object contains variables that have a String name and which values are any Java Object. The Session environment can be extended by the programmer by adding fields in the class SessionEnvironment.

Session identifier. See Session environment.

Transaction variables. We name Transaction variables the GET and POST variables associated to a HTTP Request. They are held in an object of the class TransactionVariables. These variables have a String name and a String value. In the case where the developper is sure that the String value is a boolean, an integer, etc., the class TransactionVariables provides method to extract these values. Please consult the JavaDoc of the class TransactionVariables for a list of access functions.
