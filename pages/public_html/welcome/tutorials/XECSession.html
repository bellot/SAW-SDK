A first XEC Session Binz

XML Elements is a set of classes for representing XML data in memory. The main package for these classes is elements. The base class of all XML Elements classes is XMLElement. This class is extended in many ways to provide various XHTML elements. The idea is that a page can be specified as a XML element to avoid the uncheckable litanies of PHP-like write(...) instructions as in A first PHP-like Session Binz. Moreover, XML Elements can and must be compiled. That's why we talk of XML Elements Compilation (XEC). XEC produces a Compilation object fo class Compilation. This object contains an array of bytes ready to be output to the client agent. Using XEC is most efficient and most secure way of programming a page.

In order to use XEC, the programmer has to import the four packages: 

import org.saw.elements.* ;
import org.saw.compilations.* ;
import org.saw.sessions.* ;
import org.saw.transaction.* ;

Then he must create an XML Element for the page content. For instance: 

Html page = new Html(new Head(new Title("Test3")),
                     new Body(new H1("Hello World"))) ;

Then, the XML Element must be compiled: 

Compilation compilation = page.compile() ;

The Compilation object produces its output using the Transation object: 

compilation.writeTo(transaction) ;

Example. The full code can be seen at test/XEC.java. And the Session Binz can be called at test/XEC.class.

Note that the Html element will output the two XML header lines that we have explicitely written in A first PHP-like Session Binz.

To compile this class, it is sufficient to execute the mvn install instruction in the SAW-SDK directory.
